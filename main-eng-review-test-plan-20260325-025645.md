## Affected Pages/Routes
- `/index.html` (desktop) — Full Win98 desktop with 6 app icons, taskbar, and window management
- `/index.html` (mobile) — Win98 error dialog gate + plain-text fallback content

## Key Interactions to Verify
- Double-click each desktop icon (About Me, Skills, My Projects, Internet Explorer, Contact Me, Resume.pdf) — correct window opens with expected content
- Drag any window by its title bar — moves smoothly, doesn't escape viewport
- Click minimize button — window hides, taskbar button persists
- Click taskbar button of minimized window — window restores
- Click maximize button — window fills viewport (minus 36px taskbar)
- Click maximize again — window restores to original size/position
- Click close button — window removed, taskbar button removed
- Open 3+ windows, click between them — z-order updates (clicked window comes to front)
- In Explorer: double-click a project folder — IE window opens with that project's URL
- In IE: iframe loads or shows "Page cannot be displayed" fallback with "Open in new window" link
- In Contact: click Send — mailto link triggers (or email address is visible/copyable)
- Taskbar clock — shows current time, updates

## Edge Cases
- Double-click same icon twice — should focus existing window or open second (verify behavior is intentional)
- Minimize all windows — taskbar shows all, desktop visible
- Maximize then minimize — restore should return to pre-maximize position
- Drag window to screen edge — should not disappear offscreen
- IE iframe for site that blocks embedding — fallback error page appears within 3s

## Critical Paths
- Desktop load → icons visible → double-click About Me → window opens with bio text → drag window → close window (full lifecycle)
- Explorer → double-click project → IE opens → iframe loads or fallback shows (cross-app navigation)
- Mobile viewport (<768px) → error dialog visible → OK button → plain-text fallback content visible
